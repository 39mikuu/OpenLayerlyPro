## External-secret semantics

- Rotating the session secret invalidates existing sessions and can make in-flight encrypted login-code tasks unreadable.
- Losing the configuration encryption key makes encrypted DB overrides unrecoverable.
- Standard Docker Compose now supports persistent automatic session-secret generation through #119/#120 while preserving environment/file overrides.
- The operator confirms custody and recovery/loss behavior was exercised; the final report must retain non-secret evidence metadata.

## Draft release notes

OpenLayerlyPro v1.0.0 combines manual and Stripe payments, recurring subscriptions, auditable membership lifecycle, reliable mail/tasks, secure local/S3 delivery, Markdown and multilingual publishing, AI-assisted translation, nonce-based CSP/security headers, hardened recovery, and persistent automatic Compose session-secret generation.

Upgrade from v0.1.0 requires a complete recovery point, stopped writes, pending-payment remediation, the forward migrator, and the mandatory file-safety backfill. Downgrade migrations are unsupported; rollback restores the complete pre-upgrade database/config/local archive, previous image/source, matching object-storage recovery point, and required external secrets.
