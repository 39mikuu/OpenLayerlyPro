## High-Confidence Findings

- Session tokens are generated as random values and stored only as HMAC hashes.
- Session cookies are `httpOnly`, `sameSite=lax`, and `secure` in production.
- Logout deletes the current session row and clears the cookie.
- Email login codes are HMAC-hashed, expire, have attempt limits, and are marked used inside a transaction.
- Login-code plaintext is only logged in non-production when SMTP is not configured.
- Turnstile secret key remains server-side.
- Trusted proxy parsing defaults to fail-safe behavior with `TRUSTED_PROXY_HOPS=0`.
- Process-local rate-limit buckets are cleaned by their own windows and capped.
- Admin APIs consistently use `requireAdmin` in the scanned route set.
- Fan account/payment APIs use `requireUser` where user-specific access is required.
- Payment proof files require owner or admin access.
- Content files are authorized through associated published posts and membership checks.
- Public `/api/site` returns public site info only and does not include custom footer HTML.
- Custom footer HTML is rendered only through public theme chrome and is documented as trusted admin-managed code.
- Translation provider keys are stored in encrypted config, are not returned in admin views, and AI draft generation is admin-triggered only.
- Required high-frequency indexes are present in schema and migration history.

## Known Limitations

- Uploads are read fully into memory before being written to local/S3 storage. Operators should tune `MAX_UPLOAD_SIZE_MB` and `PAYMENT_PROOF_MAX_SIZE_MB`; streaming upload is future work.
- Process-local rate limiting is suitable for single-instance deployments only. Multi-instance deployments need an external limiter such as Redis/Valkey in a future phase.
- Admin payment review lists and content lists are simple today; larger sites may need richer pagination/filtering later.
- Automatic payment providers, webhooks, and reconciliation are intentionally out of scope for v0.1.
- Plugin runtime and theme marketplace are intentionally out of scope for v0.1.
- Content moderation, comments, likes, follows, and discovery feeds are intentionally out of scope.

## Suggested Release Notes

OpenLayerlyPro v0.1.0-preview is the first open-source preview of a self-hosted single-creator membership site. It includes site setup, admin dashboard, fan email-code login, manual screenshot payment review, membership activation, permissioned posts/downloads, local and S3/R2 storage, creator-support public theme, zh/en/ja UI, content translation workflows, AI-assisted drafts, brand assets, and trusted admin-managed footer code.

OpenLayerlyPro is licensed under `AGPL-3.0-only`. Modified versions offered to users over a network must provide their corresponding source as required by the license.

Before upgrading or deploying, back up the PostgreSQL database and `/app/secrets/config-encryption-key` or the Docker `secrets` volume.
