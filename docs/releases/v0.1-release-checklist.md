# v0.1 Release Checklist

Use this checklist before tagging the v0.1 preview release.

## 1. Code Checks

- [ ] `pnpm install --frozen-lockfile`
- [ ] `pnpm test`
- [ ] `pnpm lint`
- [ ] `pnpm format:check`
- [ ] `pnpm exec tsc --noEmit`
- [ ] `pnpm build:migrator`
- [ ] `pnpm build`

## 2. Database

- [ ] Review generated migrations.
- [ ] Apply migrations in a staging environment.
- [ ] Back up the production database.
- [ ] Back up `/app/secrets/config-encryption-key` or the Docker `secrets` volume.
- [ ] Confirm rollback plan: restore database backup, restore config key, redeploy previous image/tag.

## 3. Deployment

- [ ] `docker compose up -d --build`
- [ ] `GET /api/health` returns 200.
- [ ] `GET /api/ready` returns 200.
- [ ] Reverse proxy routes only HTTPS traffic to the app.
- [ ] Cloudflare Tunnel or reverse proxy public hostname points to `http://app:3000`.
- [ ] `APP_URL` matches the public URL.
- [ ] `TRUSTED_PROXY_HEADER` and `TRUSTED_PROXY_HOPS` match the deployment edge.
- [ ] Origin port is not publicly reachable when trusting proxy IP headers.

## 4. Functional Smoke Test

- [ ] Initialize the site at `/admin/setup`.
- [ ] Log in as admin.
- [ ] Configure SMTP and send a test email.
- [ ] Configure Turnstile or confirm it is intentionally disabled.
- [ ] Fan email-code login works.
- [ ] Create or edit membership tiers.
- [ ] Create a payment method with a QR image.
- [ ] Fan submits a payment proof.
- [ ] Admin approves the payment request.
- [ ] Membership becomes active.
- [ ] Fan accesses member-only content.
- [ ] Fan downloads an authorized attachment.
- [ ] Unauthorized user cannot download member-only attachments.
- [ ] Language switching works for `zh`, `en`, and `ja`.
- [ ] AI translation can be manually triggered by admin when explicitly enabled.
- [ ] AI translation remains disabled by default.
- [ ] Custom logo/icon render on public pages.
- [ ] Custom footer code appears only on public pages.
- [ ] Custom footer code does not render in admin pages or API responses.

## 5. Security Checks

- [ ] `SESSION_SECRET` is strong and not `change-me`.
- [ ] `CONFIG_ENCRYPTION_KEY` or key file is backed up.
- [ ] SMTP, S3/R2, Turnstile, and AI provider secrets are not committed.
- [ ] File upload limits match available memory.
- [ ] Payment proof files are not publicly accessible.
- [ ] Admin routes and APIs require admin session.
- [ ] Fan APIs require user session where needed.
- [ ] No production code logs secrets, login codes, session tokens, or provider keys.
- [ ] Custom footer code warning is visible in admin settings.

## 6. Release

- [ ] Update `CHANGELOG.md`.
- [ ] Confirm README version/status text.
- [ ] Create release tag, recommended: `v0.1.0-preview`.
- [ ] Publish GitHub Release notes.
- [ ] Publish Docker image tag if applicable.
- [ ] Include upgrade notes and backup reminder.
- [ ] Include known limitations.

## Release Notes Draft

OpenLayerlyPro v0.1.0-preview is the first open-source preview of a self-hosted single-creator membership site. It includes site setup, admin dashboard, fan email login, manual screenshot payments, membership activation, permissioned content/downloads, storage configuration, creator-support theme, i18n for zh/en/ja, manual and AI-assisted translation workflows, brand assets, and admin-managed custom footer code.

Known limitations: no automatic payment providers, no plugin runtime, no comments/likes/follows, one creator per deployment, uploads are memory-bound, and process-local rate limiting is intended for single-instance deployments.
