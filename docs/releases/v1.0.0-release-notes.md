## What v1.0.0 is

OpenLayerlyPro v1.0.0 is a self-hosted, single-creator membership site combining:

- manual proof-review payments, Stripe one-time hosted Checkout, and Stripe
  recurring subscriptions, with invoice-level financial idempotency,
  reversal-first refund/dispute tombstones, provider-event inbox/dispatcher
  processing, and provider-clock-fenced reconciliation;
- an auditable membership lifecycle with per-user serialized grants and manual
  renewal reminders for non-card deployments;
- reliable transactional mail: failure classification, operator defer/dead
  states, stable Message-ID, delivery ledger, and retry visibility;
- secure local/S3 file delivery: bounded request bodies, server-authoritative
  image safety with quarantine and mandatory backfill, atomic deletion with
  complete reference checks, streaming uploads, single-range playback, and
  separate playback/download authorization;
- Markdown and multilingual publishing (zh/en/ja) with versioned translations
  and OpenAI-compatible AI draft generation under creator review control;
- nonce-based CSP and global security response headers with dynamic
  Turnstile/storage/video/integration sources;
- hardened backup/restore: archive v2 integrity plus manifest v3
  image-authoritative runtime provenance, backup-tool provenance, and
  config-key fingerprint/format validation — all fail-closed — with legacy
  schema probing, restored-task/provider-event neutralization, and DB↔storage
  convergence before serving traffic;
- automated first-boot secret provisioning: atomic `0600` `SESSION_SECRET`
  generation and atomic `CONFIG_ENCRYPTION_KEY` provisioning.

## Upgrading

Upgrade from `v0.1.0` requires, in order: a complete recovery point, stopped
writes, pending-payment conflict remediation, the one-off forward migrator,
and the mandatory file-safety backfill before normal app startup
(`docs/deployment/upgrade.md`). Downgrade migrations are not supported;
rollback restores the complete pre-upgrade DB/config/local archive, previous
image/source, matching object-storage recovery point, and external secrets.

Operators must hold `SESSION_SECRET` and `CONFIG_ENCRYPTION_KEY` outside the
application host; neither is automatically recoverable in every deployment
mode, and encrypted configuration is unrecoverable without the original key.

## Build identity

The release image is built with inline
`OPENLAYERLY_BUILD_VERSION` / `OPENLAYERLY_BUILD_COMMIT` /
`OPENLAYERLY_BUILD_TIMESTAMP` arguments; `/app/build-info.json`, OCI labels,
the image ID, and backup manifest v3 runtime provenance must agree, and no
`.env`/container environment value may override build identity
(`docs/release-v1.0-checklist.md`, section 5).

## Out of scope for v1.0

Plugin runtime, Hub official plugin, multi-instance high availability, video
transcoding/thumbnails/HLS-DASH/multipart Range, archived-translation history
viewing, and unbounded-history pagination (tracked for v1.1).
